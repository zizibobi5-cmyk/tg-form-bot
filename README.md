# Telegram-бот для приёма и публикации анкет

Бот собирает анкеты у пользователей пошагово, отправляет их в чат модераторов для проверки, и публикует принятые анкеты в Telegram-канале вместе с фотоальбомом.

## Возможности

- Пошаговое заполнение анкеты (FSM): Имя/Фамилия, Возраст/Рост, Магические способности, Характер, Биография (мин. 300 символов), Интересные факты (опционально), Место работы/должность/факультет/орден/фракции, Место проживания (опционально), Ролл, до 10 фото.
- Сбор нескольких фотографий, включая альбомы (media groups).
- Длинная анкета отправляется как медиагруппа + до 7 текстовых сообщений (по 4096 символов).
- Предпросмотр анкеты перед отправкой и кнопки «Отправить» / «Изменить» / «Отмена».
- Карточка модерации в отдельном чате с инлайн-кнопками «Принять» / «Отклонить» / «Бан».
- Кнопками модерации могут пользоваться только админы из `ADMIN_IDS` и явно добавленные модераторы (через `/addmod`). Остальные участники чата модераторов не имеют доступа.
- При отклонении модератор вводит причину, пользователь получает уведомление и предложение подать заново.
- Раздел «Задать вопрос» — переписка пользователь ↔ модератор через бота.
- Публикация принятой анкеты в канал медиагруппой с текстом.
- Антифлуд (настраиваемый интервал).
- Лимит принятых анкет на пользователя (по умолчанию 30).
- Бан пользователя (через модератора или админ-командой).
- Админ-команды: список модераторов, бан/разбан, статистика; `/addmod`, `/delmod`, `/ban` поддерживают Reply прямо в чате модераторов.

## Стек

- Python 3.11+
- aiogram 3.x
- SQLAlchemy 2 (async) + SQLite (aiosqlite)
- python-dotenv

## Установка и запуск

### 1. Создайте бота через @BotFather

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram.
2. Команда `/newbot`, придумайте имя и username.
3. Сохраните токен (выглядит как `123456:ABC-DEF...`).
4. Полезные настройки:
   - `/setprivacy` → **Disable** — чтобы бот в группе модераторов видел все сообщения (нужно для ForceReply-ответов).
   - `/setjoingroups` → **Enable**.

### 2. Создайте канал для публикации

1. Создайте Telegram-канал (публичный или приватный).
2. Добавьте бота в администраторы канала с правом «Публикация сообщений».
3. Узнайте ID канала:
   - **Приватный канал**: перешлите любое сообщение из канала в [@userinfobot](https://t.me/userinfobot) — он покажет ID вида `-100xxxxxxxxxx`.
   - **Публичный канал**: можно тоже использовать переадресацию, либо просто использовать `@username` (но в `.env` лучше всё же поставить числовой ID).

### 3. Создайте чат модераторов

1. Создайте группу в Telegram.
2. Добавьте туда бота и всех модераторов.
3. Узнайте ID группы тем же способом (переслать сообщение из группы в @userinfobot).
4. Включите бота в админы группы (иначе он не сможет писать в неё, если включен медленный режим/закрепы).

### 4. Узнайте свой Telegram ID

Напишите [@userinfobot](https://t.me/userinfobot) команду `/start` — он покажет ваш ID. Этот ID указывается в `ADMIN_IDS` в `.env`.

### 5. Установите зависимости

```bash
git clone <repo> tg-form-bot
cd tg-form-bot
python3.11 -m venv .venv
source .venv/bin/activate           # Linux/macOS
# .venv\Scripts\activate            # Windows
pip install -r requirements.txt
```

### 6. Настройте .env

```bash
cp .env.example .env
nano .env   # или любой редактор
```

Заполните как минимум `BOT_TOKEN`, `CHANNEL_ID`, `MODERATOR_CHAT_ID`, `ADMIN_IDS`.

### 7. Запустите бота

```bash
python bot.py
```

При первом запуске бот автоматически создаст файл `data/bot.db` (SQLite) и нужные таблицы.

В логах вы увидите:
```
... [INFO] __main__: Bot starting...
... [INFO] __main__: Channel: -100..., ModeratorChat: -100..., Admins: [111111111]
... [INFO] aiogram.dispatcher: Start polling
```

## Использование

### Пользователь

- `/start` — приветствие и меню.
- Кнопка **«📝 Отправить анкету»** — пошаговое заполнение → предпросмотр → отправка.
- Кнопка **«❓ Задать вопрос»** — написать вопрос модераторам.
- Кнопка **«📋 Правила»** — посмотреть правила.
- `/cancel` — отменить текущее заполнение.
- `/menu` — вернуться в главное меню.

### Модератор

В чате модераторов под каждой новой анкетой появляются кнопки:
- **✅ Принять** — анкета публикуется в канал, пользователь получает уведомление.
- **❌ Отклонить** — бот запрашивает причину отказа: модератор отвечает на сообщение бота (ForceReply). Пользователь получает причину и предложение подать заново.
- **🚫 Бан пользователя** — пользователь блокируется в боте, анкета отклоняется.

Вопросы пользователей приходят в тот же чат с кнопкой **✍️ Ответить**. Нажмите её, ответьте на сообщение бота — текст уйдёт пользователю.

### Администратор

Работают **в личке с ботом** и **в чате модераторов** (только если отправитель — админ из `ADMIN_IDS`):
- `/admin` — список команд.
- `/addmod <user_id>` / `/delmod <user_id>` — управление модераторами. В чате модераторов можно ответить (Reply) на сообщение пользователя командой `/addmod` или `/delmod` — ID подставится автоматически.
- `/mods` — список назначенных модераторов.
- `/ban <user_id> [причина]` / `/unban <user_id>` — бан/разбан в боте. Тоже поддерживают Reply: ответьте `/ban` или `/ban <причина>` на сообщение пользователя.
- `/stats` — статистика по анкетам.

**Кто может управлять анкетами** (жать «Принять / Отклонить / Бан» и «Ответить» на вопросы):
- админы из `ADMIN_IDS`;
- модераторы, явно добавленные через `/addmod`.

Все остальные участники чата модераторов получат alert «У вас нет прав модератора».

## Лимиты и настройки

Все основные параметры — в `.env`:

| Переменная               | Назначение                                          | По умолчанию |
| ------------------------ | --------------------------------------------------- | ------------ |
| `ANTIFLOOD_SECONDS`      | Минимальный интервал между сообщениями              | 3            |
| `MAX_APPROVED_PER_USER`  | Максимум принятых анкет от одного пользователя      | 30           |
| `MAX_PHOTOS` / `MIN_PHOTOS` | Допустимый диапазон количества фото в анкете    | 10 / 1       |
| `RULES_URL`              | Доп. ссылка на правила (опционально)                | -            |

## База данных

По умолчанию SQLite (`data/bot.db`). Чтобы переключиться на PostgreSQL — поменяйте `DATABASE_URL`, например:

```
DATABASE_URL=postgresql+asyncpg://user:pass@localhost:5432/bot
```

и установите дополнительно `asyncpg` (`pip install asyncpg`).

## Запуск как systemd-сервис (опционально)

`/etc/systemd/system/tg-form-bot.service`:

```ini
[Unit]
Description=Telegram form bot
After=network.target

[Service]
Type=simple
User=botuser
WorkingDirectory=/opt/tg-form-bot
ExecStart=/opt/tg-form-bot/.venv/bin/python bot.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now tg-form-bot
sudo journalctl -u tg-form-bot -f
```

## Частые проблемы

- **Бот не отвечает в группе модераторов** — проверьте, что privacy mode отключен (`/setprivacy` → Disable у BotFather) и бот добавлен в группу.
- **Не публикуется в канал** — бот должен быть админом канала с правом публикации сообщений.
- **`-1009xxxxx` not found** — у вас, скорее всего, перепутан ID. Для приватных каналов и супергрупп он должен начинаться с `-100`.
- **«Эта анкета уже обработана»** — кто-то из модераторов уже нажал «Принять»/«Отклонить» раньше.
- **Не приходит причина отказа пользователю** — пользователь должен хотя бы раз сам написать боту (нельзя писать пользователю первым, если он не запускал бота).

## Telethon юзербот для премиум-эмодзи (опционально)

Telegram **режет премиум/кастомные эмодзи** (`<tg-emoji>`) у сообщений, которые
публикуют боты в каналах. Это ограничение самой платформы; ни `copy_message`,
ни `forward_message` не помогают. Чтобы сохранять премиум-эмодзи, в репо
добавлен второй сервис — Telethon-юзербот, который входит в Telegram как
обычный пользователь с Premium и публикует анкеты в канал **от вашего имени**.

### Архитектура

```
┌──────────────────┐                       ┌──────────────────────┐
│  tg-form-bot     │   enqueue в очередь   │  tg-form-userbot     │
│  (aiogram, бот)  │ ────────────────────► │  (Telethon, юзербот) │
│                  │   channel_publications│                      │
│  • сбор анкет    │ ◄─── общая Neon БД ──►│  • публикация в      │
│  • модерация     │                       │    канал от Premium  │
│  • очередь       │                       │  • сохраняет emojis  │
└──────────────────┘                       └──────────────────────┘
```

При модерации анкеты:
- Если `USERBOT_ENABLED=false` (по умолчанию) — бот публикует сам, как раньше
  (премиум-эмодзи режутся).
- Если `USERBOT_ENABLED=true` — бот кладёт задачу в `channel_publications`,
  а юзербот забирает её и публикует от вашего имени.

Это означает, что юзербот можно выключить в любой момент (или его сервис
может временно лежать), и анкетный бот продолжит работать без него — нужно
только переключить `USERBOT_ENABLED=false`.

### Шаг 1. Получите TELEGRAM_API_ID / TELEGRAM_API_HASH

1. Откройте https://my.telegram.org → войдите под Premium-аккаунтом,
   от лица которого будут публиковаться анкеты.
2. **API development tools** → создайте приложение (название/short name
   любые, platform = Other).
3. Скопируйте `api_id` (число) и `api_hash`.

> Совет: можно создать **отдельный Telegram-аккаунт с Premium** специально
> для бота — тогда основной аккаунт не светится, и можно безопасно отдать
> SESSION серверу.

### Шаг 2. Сгенерируйте TELEGRAM_SESSION

Локально, на своей машине (не на Render):

```bash
pip install telethon
export TELEGRAM_API_ID=...
export TELEGRAM_API_HASH=...
python scripts/generate_session.py
```

Скрипт спросит номер телефона, код из Telegram (Telegram пришлёт его в
сообщении/SMS) и, если есть, облачный пароль 2FA. После успешного входа
выдаст одну длинную строку — это и есть `TELEGRAM_SESSION`. Скопируйте её
для следующего шага.

⚠️ **Безопасность**: эта строка эквивалентна логину в Telegram. Не
коммитьте её в репозиторий, не публикуйте в чате. Если строка утекла —
удалите сессию через настройки Telegram (Settings → Devices → Terminate)
и сгенерируйте новую.

### Шаг 3. Деплой на Render

В `render.yaml` уже описаны **два сервиса**: `tg-form-bot` (web)
и `tg-form-userbot` (web). При следующем деплое блюпринта Render
предложит создать новый сервис; примите.

Переменные окружения **для нового сервиса** `tg-form-userbot`:

| Переменная             | Значение                                          |
| ---------------------- | ------------------------------------------------- |
| `TELEGRAM_API_ID`      | число из my.telegram.org                          |
| `TELEGRAM_API_HASH`    | строка из my.telegram.org                         |
| `TELEGRAM_SESSION`     | строка из `scripts/generate_session.py`           |
| `CHANNEL_ID`           | тот же, что у `tg-form-bot`                       |
| `DATABASE_URL`         | тот же, что у `tg-form-bot` (Neon)                |
| `USERBOT_POLL_INTERVAL`| `5` (опционально)                                 |

И обновите переменную у существующего сервиса `tg-form-bot`:

| Переменная             | Значение                                          |
| ---------------------- | ------------------------------------------------- |
| `USERBOT_ENABLED`      | `true`                                            |

### Шаг 4. Чтобы Render Free Web Service не засыпал

Юзербот, как и бот, поднимает мини-HTTP-сервер `/health` на порту, который
Render выдаёт через `$PORT`. Добавьте URL юзербота в UptimeRobot/cron-job.org
рядом с URL бота, чтобы оба сервиса оставались разогретыми.

### Проверка

1. Откройте бота → отправьте тестовую анкету с премиум-эмодзи.
2. Модератор нажимает «✅ Принять».
3. В таблице `channel_publications` появляется строка со статусом
   `pending`.
4. Через несколько секунд юзербот публикует анкету в канал —
   премиум-эмодзи отображаются корректно.

Если что-то не так, посмотрите логи `tg-form-userbot` в Render —
в них пишутся ID задачи, ошибки Telethon и количество попыток.

### Откат

Чтобы вернуться к публикации ботом (без юзербота):

1. Render → `tg-form-bot` → Environment → `USERBOT_ENABLED=false`.
2. Render → `tg-form-userbot` → Suspend (или удалить сервис).

Бот возобновит публикацию напрямую, как раньше.
